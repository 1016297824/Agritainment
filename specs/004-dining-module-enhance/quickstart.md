# Quickstart: 餐饮模块功能增强

**Date**: 2026-05-11
**Feature**: 004-dining-module-enhance

## 前置条件

1. 项目可正常构建运行
2. 数据库已执行迁移脚本（如有新增字段）
3. 用户已登录

## 验证步骤

### Step 1: 验证预约日期独立性 (P1)

1. 进入餐饮预约页面
2. 选择"今天" + "中午"时段
3. 预约桌位 A01
4. 切换到"明天" + "中午"时段 → **验证 A01 仍显示"可预约"**
5. 切换到"今天" + "夜晚"时段 → **验证 A01 仍显示"可预约"**
6. 切换到"今天" + "中午"时段 → **验证 A01 显示"已预订"**

### Step 2: 验证重复预约拒绝

1. 在已有一个"今天中午"预约的情况下
2. 尝试预约今天中午的另一个桌位
3. → **验证系统提示"该时段已有预约"**

### Step 3: 验证预约到点餐流程 (P2)

1. 完成一个预约
2. → **验证自动跳转到点餐页面**
3. → **验证点餐页面顶部显示预约桌位信息**
4. 浏览菜品列表，点击"+"添加菜品到购物车
5. 点击"-"调整数量
6. → **验证购物车总数量和总金额实时更新**
7. 点击"下单"
8. → **验证订单创建成功，显示订单确认**

### Step 4: 验证未预约拦截

1. 没有有效预约（或取消所有预约）
2. 点击底部"点餐"导航
3. → **验证系统提示"请先完成预约"，停留在预约页面**

### Step 5: 验证边界情况

1. 对库存为0的菜品点击"+" → **验证按钮禁用，显示"售罄"**
2. 预约后取消预约，再进入点餐 → **验证提示"请先完成预约"**
3. 快速双击预约按钮 → **验证不会创建重复预约**

## 构建命令

```bash
# 后端构建
cd server ; mvn compile

# 前端构建
cd client ; npm run dev:mp-weixin
```
