# 小程序入口登录重定向

**项目名称**: 农家乐小程序 - 入口登录重定向
**版本**: V1.0
**日期**: 2026-05-07
**状态**: 待确认

---

## 1. 功能概述

### 1.1 背景

当前小程序启动后直接进入首页，不检查用户登录状态。未登录用户可以浏览首页内容，但无法使用需要认证的功能（预约、点餐、购买等），只有在主动进入"我的"页面时才能看到登录入口，导致用户可能在使用中途才发现需要登录，体验不连贯。

### 1.2 目标

在小程序启动时根据用户登录状态进行入口页面重定向：
- **已登录用户** → 正常进入首页
- **未登录用户** → 进入"我的"页面，引导完成登录

### 1.3 用户角色

| 角色 | 行为 |
|------|------|
| 未登录用户 | 启动小程序后被引导至"我的"页面，看到登录提示 |
| 已登录客户 | 启动小程序后正常进入首页 |
| 已登录员工 | 启动小程序后正常进入首页 |
| 已登录管理员 | 启动小程序后正常进入首页 |

---

## 2. 用户场景

### 场景 1：未登录用户首次打开小程序

1. 用户打开农家乐小程序
2. 系统检测到用户未登录
3. 自动跳转到"我的"页面
4. 页面展示"登录后享受更多服务"提示和"立即登录"按钮
5. 用户点击"立即登录"，进入登录页完成登录
6. 登录成功后跳转回首页

### 场景 2：已登录用户打开小程序

1. 用户打开农家乐小程序
2. 系统检测到用户已登录
3. 正常展示首页内容

### 场景 3：用户登录后再次打开小程序

1. 用户之前已登录（token 仍有效）
2. 打开小程序时系统检测到有效登录状态
3. 正常展示首页内容

### 场景 4：用户 token 过期后打开小程序

1. 用户之前登录过但 token 已过期
2. 打开小程序时本地仍有 token，但实际已失效
3. 用户进入首页后，当发起需要认证的请求时，后端返回 401
4. 请求层自动清除过期 token 并跳转登录页

## 3. 功能需求

### FR-1: 启动时登录状态检测

小程序启动时（`onLaunch`），检查用户是否已登录。

**验收标准**：
- 在 `App.vue` 的 `onLaunch` 生命周期中执行登录状态检查
- 使用 auth store 的 `isLoggedIn` getter 判断登录状态

### FR-2: 未登录用户重定向

当检测到用户未登录时，将用户重定向到"我的"页面。

**验收标准**：
- 未登录时自动跳转到 `pages/profile/index`（"我的"页面）
- "我的"页面已有未登录状态的登录提示 UI，无需额外修改
- 使用 `uni.switchTab` 进行跳转（因为"我的"是 TabBar 页面）

### FR-3: 已登录用户保持默认行为

当检测到用户已登录时，不做额外跳转，保持默认进入首页的行为。

**验收标准**：
- 已登录用户正常进入首页 `pages/index/index`
- 不产生任何额外的页面跳转或闪烁

### FR-4: 登录成功后按角色跳转首页

用户在"我的"页面完成登录后，根据用户角色自动跳转到对应首页。

**验收标准**：
- 登录成功后自动跳转到对应角色的首页
- customer/staff 角色跳转到首页（TabBar 页面），使用 `uni.switchTab`
- admin 角色跳转到管理后台（非 TabBar 页面），使用 `uni.reLaunch`

---

## 4. 非功能需求

### NFR-1: 性能

- 登录状态检查和重定向应在页面渲染前完成，避免用户看到首页后再跳转的闪烁

### NFR-2: 兼容性

- 不影响现有页面的正常功能
- 不影响后端 401 自动跳转登录页的现有机制

---

## 5. 假设与约束

### 假设

- "我的"页面（profile/index）已有的登录提示 UI 满足需求，无需修改
- 登录页（login/index）现有的登录流程无需修改
- auth store 的 `isLoggedIn` 基于 token 存在性判断，足以在启动时做初步判定
- token 过期的情况由现有 401 拦截机制处理，不需要在启动时额外验证 token 有效性

### 约束

- 微信小程序的 `onLaunch` 中无法完全阻止首页渲染，重定向可能产生短暂闪烁
- `uni.switchTab` 只能跳转到 TabBar 页面，不能携带参数

---

## 6. 成功标准

1. 未登录用户打开小程序后被引导至"我的"页面，能看到登录入口
2. 已登录用户打开小程序后正常进入首页，无额外跳转
3. 用户在"我的"页面完成登录后能顺利跳转到首页
4. 整个流程无明显的页面闪烁或卡顿
5. 不影响现有已登录用户的使用体验

---

## 7. 范围界定

### 包含

- App.vue 启动时的登录状态检测和重定向逻辑
- 登录成功后的首页跳转逻辑调整

### 不包含

- "我的"页面的 UI 修改
- 登录页面的流程修改
- Token 有效性的服务端验证
- 新增登录页面或注册流程
