## 1. 项目概述

### 1.1 项目简介

**农家乐小程序**是一个集餐饮预约点餐、产品服务售卖、种植体验于一体的微信小程序综合管理系统。

### 1.2 项目目标

- 为客户提供便捷的预约、点餐、购买、监控服务体验
- 为员工提供高效的扫码核销、服务管理工具
- 为管理员提供完善的后台管理能力

### 1.3 用户角色

| 角色             | 描述       | 登录方式     | 权限范围                     |
| ---------------- | ---------- | ------------ | ---------------------------- |
| **客户**   | 普通用户   | 手机号验证码 | 预约、点餐、购买、监控       |
| **员工**   | 服务人员   | 手机号验证码 | 扫码核销、预约管理、服务确认 |
| **管理员** | 系统管理员 | 手机号+密码  | 全部管理权限                 |

## 4. 核心功能模块

### 4.1 认证与权限模块

#### 4.1.1 功能概述

提供用户注册、登录、权限验证等功能，支持三种角色：客户、员工、管理员。

#### 4.1.2 实现技术

| 组件          | 技术                      | 文件位置                                                               |
| ------------- | ------------------------- | ---------------------------------------------------------------------- |
| JWT生成与验证 | JJWT (io.jsonwebtoken)    | `util/JwtUtil.java`                                                  |
| 认证拦截      | Spring HandlerInterceptor | `interceptor/AuthInterceptor.java`                                   |
| 权限拦截      | 自定义注解 + AOP          | `annotation/RequireRole.java` + `interceptor/RoleInterceptor.java` |
| 密码加密      | BCrypt                    | `org.mindrot.jbcrypt.BCrypt`                                         |

#### 4.1.3 实现流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         认证流程                                         │
└─────────────────────────────────────────────────────────────────────────┘

客户登录流程:
┌────────────────────────────────────────────────────────────────────────────┐
│  1. 前端: 请求发送短信验证码                                               │
│     POST /api/v1/auth/sms-code                                            │
│     └── 开发环境: 验证码固定为 123456 (dev-code配置)                        │
│     └── 生产环境: 需接入真实短信服务商                                      │
│                                                                           │
│  2. 前端: 手机号 + 验证码登录                                              │
│     POST /api/v1/auth/login                                               │
│     └── AuthService.login()                                               │
│         ├── 验证手机号格式 (11位)                                          │
│         ├── 验证验证码正确性 (5分钟有效期)                                  │
│         ├── 查询用户是否存在                                               │
│         └── 生成JWT Token返回                                              │
│                                                                           │
│  3. Token 结构:                                                           │
│     {                                                                     │
│       "sub": "userId",                                                    │
│       "role": "customer|staff|admin",                                     │
│       "isMember": true|false,                                             │
│       "exp": 604800000 (7天)                                              │
│     }                                                                     │
│                                                                           │
│  4. 后续请求: 前端在 Header 中携带 Token                                   │
│     Authorization: Bearer <token>                                         │
└────────────────────────────────────────────────────────────────────────────┘

管理员登录流程:
┌────────────────────────────────────────────────────────────────────────────┐
│  1. 前端: 手机号 + 密码登录                                                │
│     POST /api/v1/auth/admin-login                                         │
│     └── AuthService.adminLogin()                                          │
│         ├── 查询角色为 admin 的用户                                        │
│         ├── BCrypt.checkpw() 验证密码                                      │
│         └── 生成JWT Token返回                                              │
└────────────────────────────────────────────────────────────────────────────┘

注册流程:
┌────────────────────────────────────────────────────────────────────────────┐
│  1. 前端: 手机号 + 验证码注册                                              │
│     POST /api/v1/auth/register                                            │
│     └── AuthService.register()                                            │
│         ├── 验证手机号和验证码                                             │
│         ├── 检查手机号是否已注册                                            │
│         ├── 生成唯一身份码 (12位数字, generateIdentityCode)                │
│         ├── 设置默认角色为 customer                                        │
│         ├── 写入 users 表                                                  │
│         └── 返回 JWT Token                                                 │
└────────────────────────────────────────────────────────────────────────────┘

Token 验证流程 (每个请求):
┌────────────────────────────────────────────────────────────────────────────┐
│  AuthInterceptor.preHandle()                                              │
│                                                                           │
│  1. 检查是否有 Authorization Header                                       │
│     └── 没有 → 返回 40101 未登录                                          │
│                                                                           │
│  2. 解析 JWT Token                                                        │
│     ├── ExpiredJwtException → 40102 Token已过期                           │
│     ├── MalformedJwtException → 40103 Token格式错误                       │
│     ├── SignatureException → 40104 Token签名无效                          │
│     └── JwtException → 40101 Token无效                                    │
│                                                                           │
│  3. 解析成功 → 将 userId、role、isMember 放入 request attributes          │
│     └── request.setAttribute("userId", userId)                            │
│     └── request.setAttribute("role", role)                                │
│                                                                           │
│  4. RoleInterceptor 检查 @RequireRole 注解                                │
│     └── 角色不匹配 → 返回 40301 权限不足                                   │
└────────────────────────────────────────────────────────────────────────────┘
```

#### 4.1.4 核心代码

**JWT工具类** (`util/JwtUtil.java`):

- `generateToken(userId, phone, role, isMember)`: 生成JWT Token
- `parseToken(token)`: 解析并验证Token
- 签名算法: HS256
- 过期时间: 7天 (604800000ms)

**权限拦截器** (`interceptor/RoleInterceptor.java`):

- 读取 `@RequireRole({RoleEnum.CUSTOMER, RoleEnum.STAFF})` 注解
- 从 request.getAttribute("role") 获取当前用户角色
- 验证角色是否在允许列表中

餐饮模块包含桌位预约、扫码点餐、订单管理三大子功能。

#### 4.2.2 功能列表

| 功能               | 客户 | 员工 | 管理员 | 说明                                     |
| ------------------ | ---- | ---- | ------ | ---------------------------------------- |
| 查看桌位状态       | ✅   | ✅   | ✅     | 查看后三天午/晚餐的可预约桌位            |
| 创建预约           | ✅   | ✅   | ✅     | 选择日期、时段、桌位预约                 |
| 取消预约           | ✅   | ✅   | ✅     | 客户可自行取消，员工可代取消             |
| 查看预约列表       | ✅   | ✅   | ✅     | 查看自己的预约（客户）/ 所有预约（员工） |
| 扫码点餐           | ✅   | ✅   | ✅     | 扫描桌上二维码进入点餐页                 |

| 新增/编辑/删除菜品 | ❌   | ❌   | ✅     | 管理菜品信息和库存                       |

#### 4.2.3 实现流程

**桌位预约流程:**

```
┌────────────────────────────────────────────────────────────────────────────┐
│  1. 查看可用桌位                                                           │
│     GET /api/v1/dining/tables?date=2026-05-15&time_slot=lunch            │
│     └── DiningService.getTables()                                         │
│         ├── 查询所有桌位                                                   │
│         ├── 查询指定日期+时段已预约的桌位                                   │
│         └── 合并返回每个桌位的 reserved 状态                                │
│                                                                           │
│  2. 创建预约                                                               │
│     POST /api/v1/dining/reservations                                      │
│     └── DiningService.createReservation()                                 │
│         ├── 检查用户是否在黑名单 (is_blacklisted)                          │
│         ├── 检查该时段用户是否已有预约                                      │
│         ├── 检查该桌位是否已被其他人预约                                    │
│         ├── 插入 table_reservations 表 (status=pending)                   │
│         └── 发送微信订阅消息通知 (NotificationService)                      │
│                                                                           │
│  3. 取消预约                                                               │
│     DELETE /api/v1/dining/reservations/{id}                               │
│     └── DiningService.cancelReservation()                                 │
│         ├── 检查预约是否存在且状态为 pending                                │
│         ├── 检查取消人是否为预约人                                          │
│         ├── 判断是否超时取消 (<6小时为超时)                                 │
│         ├── 更新预约状态为 cancelled                                       │
│         ├── 如果超时取消, 后续定时任务会增加 no_show_count                 │
│         └── 发送微信订阅消息通知                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

**扫码点餐流程:**

```
┌────────────────────────────────────────────────────────────────────────────┐
│  1. 客户扫描桌上二维码 → 进入点餐页                                          │
│     二维码内容: QR_<table_number> (如 QR_A1)                                │
│                                                                           │
│  2. 获取活跃订单                                                           │
│     GET /api/v1/dining/orders/active?table_qr=QR_A1                       │
│     └── DiningService.getActiveOrder()                                    │
│         ├── 根据 qr_code 找到桌位                                          │
│         └── 查询该桌位状态为 active 的订单                                  │
│                                                                           │
│  3. 创建订单/加菜                                                          │
│     POST /api/v1/dining/orders                                            │
│     └── DiningService.createOrder()                                       │
│         ├── 根据 table_qr 或 reservation_id 确定桌位                        │
│         ├── 如果没有活跃订单:                                              │
│         │   ├── 创建新订单 (status=active)                                 │
│         │   └── 更新桌位状态为 dining                                      │
│         ├── 查询菜品信息 (价格、库存)                                       │
│         ├── 检查库存 (remaining_stock >= quantity)                        │
│         ├── 扣减库存 (乐观锁: version字段)                                 │
│         ├── 插入 order_items 表                                            │
│         └── 重新计算订单总金额                                              │
│                                                                           │
│  4. 结算订单 (员工操作)                                                     │
│     POST /api/v1/dining/orders/{id}/settle                                │
│     └── DiningService.settleOrder()                                       │
│         ├── 更新订单状态为 settled                                         │
│         ├── 记录结算时间 (settled_at)                                      │
│         └── 更新桌位状态为 idle                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

#### 4.2.4 核心代码

**餐饮服务** (`service/DiningService.java`):

- `getTables(date, timeSlot)`: 获取桌位状态列表
- `createReservation(userId, tableId, date, timeSlot)`: 创建预约
- `cancelReservation(userId, reservationId)`: 取消预约
- `createOrder(userId, tableQr, reservationId, items)`: 创建/加菜
- `settleOrder(orderId)`: 结算订单
- `changeTable(orderId, newTableId)`: 换桌
- `refundOrderItem(orderItemId)`: 退菜

**爽约机制:**

- 取消预约时判断: 预约时间 - 当前时间 < 6小时 → 标记为 `is_late_cancel`
- 定时任务 `ScheduledTaskService.checkNoShow()`:
  - 检查昨日超时未取消的预约
  - 增加用户 `no_show_count`
  - `no_show_count >= 3` → `is_blacklisted = true`

#### 4.3.1 功能概述

产品模块支持实体产品和服务产品的售卖，统一使用卡券形式，线下兑换。

#### 4.3.2 产品类型

| 类型               | 说明     | 示例                        | 兑换方式            |
| ------------------ | -------- | --------------------------- | ------------------- |
| **physical** | 实体产品 | 土鸡蛋、腊肉、水果礼盒      | 线下核销领取        |
| **service**  | 服务产品 | 采摘体验、钓鱼体验、BBQ套餐 | 线上预约 + 线下核销 |

#### 4.3.3 卡券系统

卡券是产品的载体，购买产品后生成卡券：

| 卡券字段     | 说明                                                           |
| ------------ | -------------------------------------------------------------- |
| code         | 12位唯一标识码                                                 |
| product_id   | 关联产品                                                       |
| user_id      | 所属用户                                                       |
| source       | 来源 (purchase购买 / gift会员赠送 / transfer转赠)              |
| status       | 状态 (available可用 / locked预约锁定 / used已用 / expired过期) |
| qr_code_data | 二维码数据                                                     |

#### 4.3.4 实现流程

## 5. 认证与权限系统详解

### 5.1 角色体系

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           角色权限模型                                       │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│                           RoleEnum 定义                                      │
│                                                                             │
│  public enum RoleEnum {                                                     │
│      CUSTOMER("customer"),   // 客户                                        │
│      STAFF("staff"),         // 员工                                        │
│      ADMIN("admin");         // 管理员                                      │
│  }                                                                          │
└─────────────────────────────────────────────────────────────────────────────┘

权限层级:
┌─────────────────────────────────────────────────────────────────────────────┐
│  ADMIN (管理员) ──────┐                                                      │
│                       │                                                      │
│  STAFF (员工) ────────┼───> 权限依次包含下层角色                              │
│                       │                                                      │
│  CUSTOMER (客户) ─────┘                                                      │
│                                                                             │
│  @RequireRole({RoleEnum.STAFF, RoleEnum.ADMIN})                              │
│      └── 员工和管理员可以访问，客户不行                                        │
│                                                                             │
│  @RequireRole({RoleEnum.CUSTOMER, RoleEnum.STAFF, RoleEnum.ADMIN})           │
│      └── 所有登录用户都可以访问                                               │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 权限注解使用示例

```java
// 只有管理员可以访问
@RequireRole({RoleEnum.ADMIN})
@DeleteMapping("/tables/{id}")
public Result<Void> deleteTable(@PathVariable Long id) { ... }

// 员工和管理员可以访问
@RequireRole({RoleEnum.STAFF, RoleEnum.ADMIN})
@PostMapping("/orders/{id}/settle")
public Result<Order> settleOrder(@PathVariable Long id) { ... }

// 所有登录用户都可以访问
@RequireRole({RoleEnum.CUSTOMER, RoleEnum.STAFF, RoleEnum.ADMIN})
@PostMapping("/reservations")
public Result<TableReservation> createReservation(...) { ... }
```

### 5.3 前端路由守卫

```javascript

// 检查页面是否需要登录
// 检查用户角色是否有权限访问该页面
```

## 12. 启动命令

```powershell
cd server
$env:JAVA_HOME="G:\Java\jdk-21"
mvn spring-boot:run
```

```powershell
cd client

npm run dev:mp-weixin
```
