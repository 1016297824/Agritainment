# Changelog

## [0.1.0.0] - 2026-05-11

### Added
- 餐饮模块：点餐功能 — 用户可在完成预约后进入点餐页面，浏览菜单、加入购物车、调整数量、提交订单
- 餐饮模块：购物车实时汇总（总数量、总金额）
- 餐饮模块：底部导航栏（预约/点餐两个tab），无预约时拦截进入点餐页
- Pinia 状态管理（dining store）：预约状态、购物车、菜品数据统一管理

### Changed
- 餐饮模块页面拆分为独立预约页和点餐页，预约成功后自动跳转至点餐页

### Fixed
- 修复桌位预约日期独立性问题：同一桌位在不同日期的预约状态互不影响
- 修复页面无法加载问题
- 售卖完的菜品"+"按钮自动禁用；过期预约自动过滤
- 预约/下单按钮增加防重复提交保护；下单失败时购物车数据不丢失

### Security
- 点餐页面无有效预约时显示引导提示并阻止进入
